Routing

Hyper Angular is having routing setup based on Angular-Router. The pre configured routing would make your job easy when dealing with any routing related changes. The routing is setup in a way to support lazy loading as well.

You can easily add, change or remove any route by simply making changes described below:

  1. Add the entry for your component in main application routing src/app/app.routes.ts E.g.

    { path: '', component: PrivateLayoutComponent, loadChildren: () => import('./pages/pages.route').then((mod) => mod.PAGE_ROUTES), canActivate: [AuthGuard], },

    Each of these properties are explained below:

    • path - relative path to component
    • component - load layout component to all pages
    • loadChildren - module to load on this route
    • canActivate - set of Guards which decides whether route can be actiavted or not. It restrict the non authenticated user to access this module.

  2. Declare path for your components inside your routing file src/app/pages/pages.route.ts E.g.

    { path: 'analytics', component: AnalyticsComponent, data: { title: 'Analytics' }, }

    Each of these properties are explained below:

    • path - relative path to component
    • component - component to load on this route
    • data - data to set title
Note that you don't need to restart the development server in order to see the menu changes getting in effect

Optionally, you might want to add the menu entry in respective layout menu component.
